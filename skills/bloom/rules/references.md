# Choosing references

References are the strongest on-brand anchor — they pin the brand's real visual language better than text can.

Pick them yourself, but make every pick intentional: attach a reference only if you can say in one line what it does for *this* image. If you can't name the reason, drop it — that bar, not a fixed count, decides how many you attach.

Run the procedure:

1. **Search the subject, not the format.** `bloom_search_user_images` is semantic: query what's in the frame ("pour-over coffee," "dashboard on a laptop"), not "Instagram post." Run a few concept queries and pool the results — one rarely surfaces the best candidates.
2. **Skip the noise.** Logos and partner lockups, off-brand or dated content, watermarked / platform-overlay / third-party screenshots, team and event photos (unless people are the subject). The model reproduces whatever you attach.
3. **None is a valid answer — both ways.** If nothing fits, attach nothing and lean on the brand layer; a bad reference is worse than none. But don't over-reject a genuine on-brand match.
4. **Exact reproduction is opt-in.** If the user explicitly asks for something in a reference to be reproduced exactly, call out that specific reference in the prompt so the model copies it, not just its style. Otherwise, don't.
